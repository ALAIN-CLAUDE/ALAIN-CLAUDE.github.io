Project : Apex Webservices and Rest APIs For Ev Company

Ev car manufacturer Malingo motors with headquarter Org in USA has 4 supply branches (Canada, germany, China , Belgium). These branches deliver cars to customers and a custom object call 'Ev deliveries' is used to store their data . At the end of each day the results of all new deliveries must be updated in headquarter org from its child orgs. Branch deliveries custom object is used to store data in Headquarter Org.


  1. Create a Connected App In child Org

    • Go to Setup,App Manager and New connected App
    • Fill in the required fields(name,contact email). In our case Ev Branches App
    • Enable OAuth Settings
    • Provide a callback Url.Use the url of the headquarter Org as in this example
    • Select OAuth Scopes.This determines the type of access you want to give to the third party api and to this environment. Click save and continue. Within 2-10 minutes changes will take effect on server
    • Our consumer key and consumer secret are automatically generated. We will use these in our Headquarter Org and can also be used to get our security token

  2. Exposing class for Ev deliveries Data as a REST Service


    • In Ev Branches Org create an apex class with our @HttpGet method.
