Lightning web components are custom HTML elements built using HTML and modern JavaScript.

  • It is light in weight and offers better compatibility with browsers and enhanced security.
  • Resusable components

Project : LWC,Apex logic to support multiple addresses for Client

CASE : My Company want to support multiple addresses for customer as separate object. In the same time, we want to keep the standard SF address fields on the Account page layout and also require these addresses to be stored as separate entities for account.

  1. Address can be Billing or Shipping. Address can be marked as main.
  2. An apex logic is required
    • If user creates a new account and at least one field for Billing address isn’t empty system must automatically create a new Main Billing Address. The same logic should be applied for Shipping Address
    • One account can have only one main billing address and only one main shipping address and If user updates Account Billing Address field changes should be automatically applied to Main Billing Address. Same logic for Shipping Address
    • If user updates Main Billing Address changes should be automatically applied to Account Billing Address. The same logic should be applied for Main Shipping Address
    • User should be able to create new address billing/shipping address and mark it as main.
  3. Create set of LWC components to display list of account addresses on the Account Lightning Page.
    • Component should show following fields: address name, main flag, address type and Component should provide an ability to open specific address for view(standard SF page)
    • Component should provide an ability to create, delete existing address, modify existing address and filter addresses by type and by flag ‘main’
